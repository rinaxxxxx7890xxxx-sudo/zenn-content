# サーバーのIPアドレスは「固定」にするのが鉄則

普段、みなさんがスマホやPCでWi-Fiに繋ぐとき、IPアドレスを意識して接続することはほぼ無いと思います。
これは、実は裏側でwi-fiルーターが自動で「あなたは今日 192.168.1.23 ね！」ってIPアドレスをあなたのスマホに自動で配っているからです。
この自動配布の仕組みを **DHCP** と呼びます。

![イメージ画面08](/images/image08.png)

ここまで読むと、「DHCPめっちゃ便利じゃん。サーバーも全部自動でいいじゃん」と思うかもしれません。
しかし、DHCPはあくまで「空いている部屋をローテーションで貸し出す」仕組みです。
前回とは違うIPアドレスに変わってしまう可能性が常にあります。
え〜常に変わったら何があかんねんと思った人、いますよね。

結論から言うと、「自分から通信を始めるか（スマホ）」、それとも「相手からの通信を待っているか（サーバー）」という役割の違いがあるので
**サーバーのIPは常に固定** しなければいけません。


# 実際にサーバーへ設定してみる
「固定で設定する」と言っても、具体的にどうやるねんというあなたへ。
サーバーのOS（linuxやWindowsServerなど）によってやり方は異なりますが、基本的には以下のような「設定ファイル」と呼ばれるテキストファイルに、直接数字を書き込みます。

Linuxサーバー（Ubuntuなど）では、ターミナルから設定ファイルを編集します。
```bash
user@ubuntu-server:~$ sudo nano /etc/netplan/01-netcfg.yaml
```
※ 設定ファイルの場所や書き方はOSやバージョンによって異なります。
そのため、実際行う際は使用しているサーバーOSによって「"サーバーOS名" 固定IP 設定」などで調べてください。

```bash
# よくあるLinuxサーバーUbuntuでのイメージ
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: false
      addresses:
        - 192.168.1.10/24
      gateway4: 192.168.1.1
```

設定を書き換えたあと、以下のコマンドで反映します。
```bash
sudo netplan apply
```
実際の運用ではDNSサーバー設定なども行いますが、今回は固定IPの仕組みを理解するため省略しています。

このように、
* 自動割り当て（DHCP）をオフにする
* 決めたIPアドレスとサブネット（/24）を直接書き込む

という作業を行うことで、サーバーにIPアドレスがカチッと固定されます。

## レンタルサーバー（さくらやエックスサーバーなど）の場合はどうなるの？と思ったあなたへ

ここまで読んで、「サーバーのIPアドレスは固定が鉄則！」ということは分かったと思います。
でも、筆者は学習中に一つの疑問が浮かんでしまいました。

「ブログを開設するときに、ConoHa WINGとかエックスサーバーを契約したけど、**IPアドレスを固定する設定なんて一度もやってないぞ……？**」と。

結論から言うと、**「レンタルサーバーでもIPアドレスは完全に固定されている（ただし、ユーザーが自分で設定する必要はない）」** というのが答えです。

どういうことなのか、これまで通りの「家やお店」の例えを使ってスッキリ解決していきましょう。

## レンタルサーバーは「巨大なシェアオフィスビル」
自分でサーバーのOSから組み立てる（AWSや社内サーバーなど）のが「一軒家を建てる」ことだとすれば、
レンタルサーバーは **「巨大なシェアオフィスビルの１部屋」を借りる** ようなイメージです。

#### ① IPアドレスはすでに「建物（運営会社）」が固定してくれている
レンタルサーバーの運営会社は、すでに世界に一つしかない固定のIPアドレス（外線）を、そのサーバーに設定してくれています。

あなたが新しくブログを開設するためにレンタルサーバーを契約したとき、裏では **「できあがっているシェアオフィスビルの、1つの部屋（フォルダ）を割り当てられた」** 状態になります。

そのため、あなたがわざわざ設定ファイルを書き換える必要はありません。
最初から固定された住所のビルに入居するので、**勝手に固定されている**んです。

#### ② 実は「他のたくさんのWebサイト」と同じIPアドレスを共有している
ここが一番の驚きポイントです。
レンタルサーバーの場合、**「あなたのブログ」も「隣の知らない人のブログ」も、実は全く同じ1つのIPアドレス（住所）を使っています。**

```text
【レンタルサーバーのビル（IPアドレス：203.0.113.50）】
  ├─ 3階 A号室：あなたのブログ（example.com）
  ├─ 3階 B号室：Aさんの趣味ブログ
  └─ 3階 C号室：Bさんの会社のサイト
```

「えっ？前の章で『同じIPアドレスを2つの機器に設定したら競合して大惨事になる』って言ってたじゃん！」と思いましたよね。

つまりこういうことです。

![イメージ画面09](/images/image09.png)

なお、レンタルサーバーでは、サーバー自体の管理権限を持っていないことが多いです。
そのため、ソフトウェアの自由なインストールや、OSレベルの設定変更ができない場合があります。
Linuxでは、このような管理者として操作できる権限を「sudo権限（管理者権限）」と呼びます。

つまりレンタルサーバーは、「部屋（Webサイトを置く場所）」は借りられるけど、「建物全体の管理」はできないイメージです。


#### VPSサーバー(さくらのVPS、ConoHa等)やクラウドサーバー(AWS等)に関して
レンタルサーバーでは、「設備付きのシェアオフィスを借りる」ようなイメージでした。

すでに環境が整っている代わりに、勝手に壁を壊したり、大きな設備変更はできません。

一方、VPSサーバーやクラウドサーバーは、「自分専用の部屋やオフィスを借りる」イメージです。

OSやソフトウェアを自分でインストールできるため、サーバーの中をかなり自由にカスタマイズすることが出来ます。
**使用用途に合わせてサーバー選定する際は、この辺を知っておくと良いかもしれません。**


# ⚠️ 割り当て時の注意点：IPアドレスの「競合」を防ごう

 途中脱線しましたが、話を戻します。
 サーバーにIPアドレスを割り当てるとき、一番やってはいけないミスが「IPアドレスの競合」です。

1つのネットワーク内で、全く同じIPアドレスを2つの機器に設定してはいけません。
もし、新しく作った別のサーバーに、間違えて同じ 192.168.1.10 を設定してしまうと……

通信　　　　「192.168.1.10さん、データを届けにきました！」
サーバー１　「はい、私です」
サーバー２　「いや、俺だけど？」
通信　　　　「えっ、どっち……！？（混乱）」 
➔ 通信が不安定になったり、正常に通信できなくなります。


これを防ぐために、「どのIPアドレスを、何の機器に割り当てたか」をExcelや管理ツールで台帳管理するといいかもしれません。
実務では、まず「そのIPアドレスがすでに使われていないか？」を確認してから、空いている番号を割り当てるようにしましょう。


# まとめ

今回は、ネットワークで決めたIPアドレスの範囲の中から、実際にサーバーへIPアドレスを割り当てる方法について解説しました。

ポイントを整理すると、こんな感じです。

- サーバーは「相手からアクセスされる側」なので、IPアドレスを固定する必要がある
- DHCPは便利だが、IPアドレスが変わる可能性があるためサーバー用途には基本向かない
- サーバーでは、固定IPを設定できる
- レンタルサーバーでは、運営会社側がすでに固定IPを設定してくれている
- VPSやクラウドサーバーでは、自分で自由に設定・管理できる
- IPアドレスは「空いていそう」で決めるのではなく、管理しながら慎重に割り当てる

IPアドレス設定は、インフラ構築の中でも最初に触れる基本操作の1つです。
最初は難しく感じるかもしれませんが、「ネットワーク上の住所を決める作業」と考えるとイメージしやすくなります。
