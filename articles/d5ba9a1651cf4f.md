# はじめに
前回の記事で、Linuxサーバーの固定IPを設定するときに、謎の英語と数字が書かれたテキストファイルが登場しましたよね。

「何これ、呪文みたいで怖い……」と思った方もいるかもしれません。
でも大丈夫です。このファイルに書かれていることは、実は**「サーバーの自己紹介カード」**のようなものなんです。

今回は、この設定ファイルを1行ずつ細かく解剖して、何が書かれているのかを日本一わかりやすく解説します！

---

# 解剖する設定ファイルはこちら

Linux（Ubuntuなど）でよく使われる、最新のネットワーク設定（Netplan）の記述です。

```yaml
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: false
      addresses:
        - 192.168.1.10/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1

```

一見すると、前回より行数が増えて複雑に見えますが、やってることは上から順番に「ネットワークの設定をします！」「有線LANの、eth0って名前のポートの設定はね……」と、設定を絞り込んでいるだけなんです。

では、1行ずつ見ていきましょう！

---

# 1行ずつの詳しい解説

### ① 設定の「枠組み」を決める（1〜3行目）

```yaml
network:
  version: 2
  ethernets:

```

* **`network:`**
「ここから下にネットワークの設定を書きますよ！」という、大きな宣言です。
* **`version: 2`**
設定ファイルの「書き方のルール（バージョン）」を指定しています。「今はバージョン2のルールで書いてるから、そのつもりで読み取ってね」とLinuxに伝えています。
* **`ethernets:`**
「ここからは有線LAN（イーサネット）の設定をします」という意味です。

---

### ② 設定する「差込口（ポート）」を指定（4行目）

```yaml
    eth0:

```

* **`eth0:`（イーサ・ゼロ）**
サーバーのパソコン本体の後ろにある、**「LANケーブルを挿す穴（ネットワークカード）」の名前**です。
パソコンに「1番目の有線LANポートの設定をするよ」と教えてあげています。
※サーバーの機種によっては `ens33` や `enp3s0` のような別の名前になっていることもあります。

---

### ③ 自動割り当て（DHCP）を拒否する（5行目）

```yaml
      dhcp4: false

```

* **`dhcp4: false`**
「IPアドレスの自動配布機能（DHCP）」を使うかどうか、の設定です。
今回はサーバーなので、勝手にIPアドレスが変わったら困ります。だから、「false（いいえ、自動配布は使いません！）」と答えて拒否しています。

---

### ④ 自分の「住所」を決める（6〜7行目）

```yaml
      addresses:
        - 192.168.1.10/24

```

* **`addresses:`**
「このサーバーのIPアドレスはこれです」と指定する項目です。
ステップ1で決めた、このサーバーの固定IPアドレスとサブネットマスク（壁の位置）を書いています。
「私の住所は『192.168.1』という町の、『10番』の部屋ですよ！」と、自分自身の住所をここでカチッと決めています。

---

### ⑤ 外の世界への「道（ルート）」を教える（8〜10行目）

```yaml
      routes:
        - to: default
          via: 192.168.1.1

```

* **`routes:` / `- to: default` / `via: 192.168.1.1**`
古い解説書だと `gateway4: 192.168.1.1` と書かれていることが多いのですが、最新のLinux（Netplan）ではその書き方は非推奨（古いルール）になりました。現在は、この `routes` という書き方が推奨されています。
一言で言うと「ルーター（外の世界へ繋がるドア）の場所」を教えています。
「`default`（宛先がわからない外の世界すべて）」に行きたいときは、`via`（〜を経由して）、`192.168.1.1` にいるルーター先輩にデータを渡しなさい、という道案内（ルート）を指示しています。

---

### ⑥ 数字を文字に変換してくれる「通訳の連絡先」を登録（11〜14行目）

```yaml
      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1

```

* **`nameservers:` / `addresses:**`
これはDNSサーバー（ネームサーバー）のIPアドレスを指定する項目です。
ネットの世界の「電話帳」であり、文字（google.comなど）を数字（IPアドレス）に変換してくれる通訳さんのような存在です。
* **`- 8.8.8.8` / `- 1.1.1.1**`
今回は、GoogleやCloudflareという大企業が「誰でも自由に使っていいよ！」と公開してくれている、とても有名で信頼できる通訳さんの住所（パブリックDNS）を登録しています。
ここに通訳さんの住所を書いておかないと、サーバーは「固定IPは設定できたけど、google.comの住所（数字）がわからないから、外のサイトにアクセスできない！」という状態になってしまいます。

---

# ⚠️ インフラ初学者が100%ハマる「インデント」の罠

このファイルを自分で書き換えるとき、**絶対にやってはいけない超重要注意点**があります。
それが「スペース（空白）の数」です。

このファイルは「YAML（ヤムル）」という形式で書かれており、スペースを使って「親子関係」を表しています。

```text
❌ 間違い（左端が揃ってしまっている）
network:
version: 2

⭕ 正しい（versionの前にスペースが2つある）
network:
  version: 2

```

特に今回追加した `routes` や `nameservers` の中身は、段々畑のようにスペースの数が細かく決まっています。
このスペースの数が1つでもズレていたり、タブキー（Tab）を使ってスペースを入力したりすると、Linuxサーバーは「書き方がおかしいので読み込めません！」とへそを曲げてエラーになります。

「設定は合っているはずなのに、なぜか動かない……」という時は、ほぼ100%このスペースの罠に引っかかっています。編集するときは慎重にスペースを打ち込みましょう！

# まとめ

最新の書き方で1行ずつ分解してみると、サーバーが外の世界と通信するために必要な要素がすべて詰まっていることが分かりましたね。

1. **`dhcp4: false`** で自動配布をストップ
2. **`addresses`** で自分の住所（固定IP）を名乗る
3. **`routes`** で外の世界へのドア（ルーター）の場所を覚える
4. **`nameservers`** で文字と数字を変換してくれる通訳さん（DNS）を登録する

この4つが揃って、初めてサーバーは一人前としてインターネットの世界に飛び出すことができます。

次回はいよいよステップ3！
今回設定した「通訳さん（DNS）」が、裏側で一体どんな大活躍をしているのか、「ドメインとDNSの仕組み」についてさらに詳しく掘り下げていきます。
