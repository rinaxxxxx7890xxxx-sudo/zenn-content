# はじめに
前回の記事で、Linuxサーバーの固定IPを設定するときに、謎の英語と数字が書かれたテキストファイルが登場しましたよね。

「何これ、呪文みたいで怖い……」と思った方もいるかもしれません。
でも大丈夫です。このファイルに書かれていることは、実は**「サーバーの自己紹介カード」**のようなものなんです。

今回は、この設定ファイルを1行ずつ細かく解剖して、何が書かれているのかを日本一わかりやすく解説します！

---

# 解剖する設定ファイルはこちら

Linux（Ubuntuなど）でよく使われる、ネットワーク設定（Netplan）の記述です。

```yaml
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: false
      addresses:
        - 192.168.1.10/24
      gateway4: 192.168.1.1

```

一見すると複雑ですが、実は上から順番に「ネットワークの設定をします！」「有線LANの、eth0って名前のポートの設定はね……」と、設定を絞り込んでいるだけなんです。

では、1行ずつ見ていきましょう！

---

# 1行ずつの詳しい解説

### 1〜3行目：設定の「枠組み」を決める

```yaml
network:
  version: 2
  ethernets:

```

* **`network:`**
「ここから下にネットワークの設定を書きますよ！」という、大きな宣言です。
* **`version: 2`**
設定ファイルの「書き方のルール（バージョン）」を指定しています。「今はバージョン2のルールで書いてるから、そのつもりで読み取ってね」とLinuxに伝えています。
* **`ethernets:`**
「ここからは有線LAN（イーサネット）の設定をします」という意味です。（もしWi-Fiならここが `wifis:` になったりします）。

---

### 4行目：設定する「差込口（ポート）」を指定する

```yaml
    eth0:

```

* **`eth0:`（イーサ・ゼロ）**
サーバーのパソコン本体の後ろにある、**「LANケーブルを挿す穴（ネットワークカード）」の名前**です。
パソコンに「1番目の有線LANポートの設定をするよ」と教えてあげています。
※サーバーの機種によっては `ens33` や `enp3s0` のような別の名前になっていることもありますが、役割は同じです。

---

### 5行目：自動割り当て（DHCP）を拒否する

```yaml
      dhcp4: false

```

* **`dhcp4: false`**
前回の記事で大活躍した「IPアドレスの自動配布機能（DHCP）」を使うかどうか、の設定です。
今回はサーバーなので、勝手にIPアドレスが変わったら困りますよね。だから、「false（いいえ、自動配布は使いません！）」と答えて拒否しています。

---

### 6〜7行目：自分の「住所」を決める

```yaml
      addresses:
        - 192.168.1.10/24

```

* **`addresses:`**
「このサーバーのIPアドレスはこれです」と指定する項目です。
* **`- 192.168.1.10/24`**
ステップ1でみんなで決めた、このサーバーの固定IPアドレスとサブネットマスク（壁の位置）です。
「私の住所は『192.168.1』という町の、『10番』の部屋ですよ！」と、自分自身の住所をここでカチッと決めています。

---

### 8行目：外の世界への「出入り口」を教える

```yaml
      gateway4: 192.168.1.1

```

* **`gateway4: 192.168.1.1`**
専門用語で「デフォルトゲートウェイ」と呼ばれるもので、一言で言うと「ルーター（外の世界へ繋がるドア）のIPアドレス」です。
サーバーが「他のパソコンにデータを送りたいな」「インターネットのサイトを見たいな」と思ったとき、どこを通って外に出ればいいのか迷ってしまいます。
そこで、「外に出たいときは、`192.168.1.1` にいるルーター先輩にデータを渡せば、外の世界に繋いでくれるよ」と教えてあげているのです。

---

# ⚠️ インフラ初学者が100%ハマる「インデント」の罠

このファイルを自分で書き換えるとき、**絶対にやってはいけない超重要注意点**があります。
それが「スペース（空白）の数」です。

このファイルは「YAML（ヤムル）」という形式で書かれており、スペースを使って「親子関係」を表しています。

```text
❌ 間違い（左端が揃ってしまっている）
network:
version: 2

⭕ 正しい（versionの前にスペースが2つある）
network:
  version: 2

```

このスペースの数が1つでもズレていたり、タブキー（Tab）を使ってスペースを入力したりすると、Linuxサーバーは「書き方がおかしいので読み込めません！」とへそを曲げてエラーになります。

「設定は合っているはずなのに、なぜか動かない……」という時は、ほぼ100%このスペース（インデント）の罠に引っかかっています。編集するときは慎重にスペースを打ち込みましょう！

# まとめ

一見難しそうなコードも、分解してみると意外とシンプルでしたね。

1. **`dhcp4: false`** で自動配布をストップして、
2. **`addresses`** で自分の住所（固定IP）を名乗り、
3. **`gateway4`** で外の世界へのドア（ルーター）の場所を覚える。

この3つが、ネットワーク設定の基本中の基本です。
