# IPアドレスとは
IPアドレスは、ネットワーク上の機器を識別するための住所のようなものです。

例えば次のようなイメージです。

| 機器 | IPアドレス |
|---|---|
| ルーター | 192.168.1.1 |
| Webサーバー | 192.168.1.10 |
| PC | 192.168.1.100 |

ネットワーク上では、同じIPアドレスを複数の機器に設定することはできません。
ここで、うちと同じIPじゃん！と疑問に思う方がいるかもしれないので補足です。
IPアドレスには、内線と外線があります。

# サブネットとは何か
サブネット（Subnet）は、**1つの大きなネットワークを小さく分割したネットワーク** のことです。

例えば会社に1000台のPCがあるとします。

全部を1つのネットワークにするとこうなります。

```text
社内ネットワーク
├─ PC1
├─ PC2
├─ PC3
├─ ...
└─ PC1000
```

機器が増えると、
通信量が増える、障害の影響範囲が広がる、管理しにくくなる という問題が出ます。

そこで分割した方がいいんじゃね？で出てきたのが、サブネットです。

```text
社内ネットワーク
├─ 開発部
│   ├─ PC1
│   ├─ PC2
│   └─ PC3
│
├─ 営業部
│   ├─ PC4
│   ├─ PC5
│   └─ PC6
│
└─ サーバー用
    ├─ Webサーバー
    └─ DBサーバー
```

### そのサブネットは何のために使うんですかい？というあなたへ
ここが実務で非常に重要です。
サブネットは単にネットワークを分割するためだけではありません。
主な目的は次の4つです。

### ① 通信量を減らす
同じネットワーク内では、機器同士が様々な情報をやり取りしています。

例えばPCが、「プリンターどこ？」「ファイルサーバーどこ？」のような問い合わせを出します。

ネットワークが大きいと、
```text
1000台全部
↓
全機器へ通知
```
となり、不要な通信が増えます。
ここでサブネット登場です。

サブネットで分割すると
```text
営業部：100台
開発部：100台
経理部：50台
```
このように、必要な範囲だけに通知できます。

---
### ② 障害の影響範囲を小さくする
例えば、営業部ネットワークで障害が起きた場合の話です。
サブネットがない場合：
```text
障害
↓
社内全体に影響
```

サブネットがある場合：
```text
障害
↓
営業部のみ
```
範囲を絞ることで、被害を最小限に限定できます。

---
### ③ セキュリティを強化する
サーバーを社員PCと同じネットワークに置くと危険です。
悪意のあるアクセスやマルウェア感染時に、サーバーまで影響する可能性があります。
マルウェア感染については、google先生に聞いてください。

例えば会社の中に、
社員が使うPC
Webサーバー
DBサーバー
が全部同じネットワークに置かれていたとします。

```text
192.168.1.0/24
├─ 社員PC
├─ Webサーバー
└─ DBサーバー
```
一見するとシンプルですが、これ実はめちゃ危険！です。

#### なぜ危険なの？と思ったあなたへ
例えば、社員PCの1台がマルウェア感染したとします。
すると、そのPCから同じネットワーク内にある機器へアクセスしやすくなります。

```text
感染した社員PC
        ↓
同じネットワークを探索
        ↓
DBサーバー発見
        ↓
不正アクセス
```
つまり、**社員PCの問題が「サーバー全体の問題」へ発展する**可能性が出てきます。

## ④ 管理しやすくする
用途ごとにネットワークを分けると、見ただけで用途がわかります。
例：
| 用途 | サブネット |
|---|---:|
| 社員PC | 192.168.1.0/24 |
| サーバー | 192.168.2.0/24 |
| 開発環境 | 192.168.3.0/24 |
| プリンター | 192.168.4.0/24 |

IPアドレスを見るだけで、192.168.2.10 → サーバー用ネットワークと判断することができます。


# IPアドレスの範囲とは
管理するネットワーク内で利用できるアドレスの範囲を事前に決定します。

```bash
192.168.1.0/24
```
この「/24」はサブネットマスクと呼ばれる設定です。

最初に見ると、`/24`って何？という疑問が出てくると思います。

# `192.168.1.0/24` の意味
これはCIDR（サイダー）表記です。
一言で言うと**そのネットワークの中に、何個のIPアドレスが含まれているか**を短く表す書き方のことです。

この「24」という数字は、IPアドレス（32ビット）のうち、「左から24番目までがネットワークの名前（町名）ですよ」ということを意味しています。
残りの8ビット分（32 - 24 = 8）が、そのネットワーク内で使える「個別の部屋番号」になります。

ちょっと小難しいなと思ったあなたへ。すみません、分解して説明します。

`1. IPアドレスを「0と1」に分解してみる`
IPアドレスは、数字が4つに区切られています。

これは元々、「8マス（8ビット）の箱が4つ並んでいる」状態です。全部で8×4 = 32マスあります。
例：**192.168.1.0**

![イメージ画面06](/images/image06.png)

画像のIPv4以外に、IPv6というアドレスも存在します。
ただ主流となっているのはIPv4のため、ここでは詳細は割愛します。
違いについて、簡単に記述しますが、フーンと流していただいてOKです。次に進みます。

| 項目 | IPv4 | IPv6|
| :--- | :--- | :--- |
| **文字の種類** | 数字だけ（10進数） | 数字 ＋ アルファベット（16進数） |
| **区切り文字** | ドット（ . ）で4つに区切る | コロン（ : ）で8つに区切る |
| **実際の例** | 192.168.1.100 | 2001:0db8:85a3:0000:0000:8a2e:0370:7334 |
| **ビット数** | 32ビット | 128ビット |
| **作れる住所の数** | 約43億個 | 約340兆の1兆倍の1兆倍（ほぼ無限） |


----

`2. 「/24」は、左から24番目に壁を立てる`
ここで「/24」の登場です。「左から24マス目に壁（境界線）を引いてね」という意味になります。

数えてみましょう。
・1つ目の塊（192）で8マス
・2つ目の塊（168）で8マス（合計16マス）
・3つ目の塊（1）で8マス（合計24マス）

つまり、3つ目の塊の後ろにピタッと壁が立つことになります。

| 【固定された町名（24マス）】| 【自由に変えられる部屋番号（8マス）】|
|---|---|
| 11000000 . 10101000 . 00000001 | 01100100 |
| （192 . 168 . 1） | （100） |

/24により、この壁より左側の「192.168.1.」は、このネットワークにいる全員が絶対に同じ数字を使わなければいけませんというルールが敷かれます。

----
`3. 残った8マスで「256部屋」が作れる`
さて、壁の右側には、自由に使えるマスが「8マス」残されました。
この8マスの「0と1の組み合わせ」が、そのまま部屋番号になります。
たとえば
一番最初の部屋：00000000 （10進数だと 0）
ー次の部屋：00000001 （10進数だと 1）
ーその次の部屋：00000010 （10進数だと 2）
〜
ーあなたのPCの部屋：01100100 （10進数だと 100）
〜
一番最後の部屋：11111111 （10進数だと 255）

10進数で0から255までを全部数えると、ぴったり256個になります。
**/24**で作成できる組み合わせは、全部で256通りということになります。

`4. 具体例：もし「/25」にしたらどうなるんだい？`
もし壁の位置を1つ右にずらして「/25」にしたらどうなるでしょうか？
範囲が1マス増えて、部屋番号に使えるマスが「7マス」に減ってしまいます。

ハ〜？？と思った、そこのあなた。
具体例を画像で説明します。

![イメージ画面07](/images/image07.png)

町名の範囲（/24→/25）が1マス増えて、部屋番号に使えるマスが「7マス」に減ってしまいます。
7マスの組み合わせは、0000000 から 1111111 までの128通りしかありません。

つまり、「**/の後の数字が大きくなると、壁が右に移動して使える部屋数が半分（256個→128個）に減る**」ということになります。

----

`💡覚えておくべき「お約束」`
「じゃあ、/24の場合は192.168.1.0 から 192.168.1.255 までの256個、全部IPアドレスの選定時に割り当てていいんだね！」と言いたいところですが、実は最初と最後の2つは使えないというルールがあります。
```text
192.168.1.0（一番最初）：ネットワーク自体を指す名前（ネットワークアドレス）

192.168.1.255（一番最後）：その町のみんな全員に同時にデータを送るための専用番号（ブロードキャストアドレス）
```
/24にて作成できる組み合わせは256通りでしたが、使用できるのは上記２つを除いた254通りです。
そのため、実際にWebサーバーやPCに設定できるのは、間の .1 〜 .254 までの254個になります。
※/25やそのほかについても同様の共通ルールです。

サブネットの使えるアドレス数については、以下サイトに分かりやすく記述しておりましたので紹介します。
https://www.ahref.org/subnet

# まとめ

長々とお疲れ様でした！
最後にもう一度、この記事の要点をフワッと頭に入れておきましょう。

1. **IPアドレスは「住所」**：同じネットワーク内で被っちゃダメ。(ただしグローバルとプライベートがある)
2. **サブネットは「部署分け」**：分けることで、通信が軽くなり、トラブルが広がらず、セキュリティもUPする。
3. **`/24` などの数字は「壁の位置」**：数字が大きくなるほど壁が右にズレて、使える部屋の数が**半分、また半分**と減っていく。
4. **最初と最後は使えない**：/24の場合`192.168.1.0` と `192.168.1.255` は特別な役割があるから、PCやサーバーには使っちゃダメ（使えるのは `.1` 〜 `.254` まで）。
