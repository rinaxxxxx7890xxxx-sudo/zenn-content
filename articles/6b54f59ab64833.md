# 機能要件
1. ミッション管理機能
* ユーザーは、以下2種類のミッションを登録・編集・削除できます。
##### 日のミッション
日ごとに達成するミッションです。
例:
- 9:00 バイトへ出発
- 15:00 寄り道せず帰宅
- 22:30 明日の準備　　などなど
##### 月のミッション
月単位で達成する大きな目標です。
例:
- 今月10冊本を読む
- 今月20日ランニングする
- 今月中にポートフォリオを完成させる などなど

2. 通知機能
* 日のミッションに設定された時刻になると、iPhoneにローカル通知を送信。
* 通知をタップするとアプリを開き、対象ミッションの達成・未達成を記録。

3. 実行ログ記録機能
* 日のミッションについて、日ごとの実行結果を記録。
記録する内容:
- 対象ミッション
- 予定日
- 予定時刻
- 結果
- 回答日時

結果は以下の3種類とします。
- 未回答
- 達成
- 未達成


4. 月次進捗記録機能
* 月のミッションについて、対象月ごとの進捗を記録します。

5. 可視化・分析機能
* 以下の内容を画面上で確認できる機能。

###### 日次ミッションの可視化
- 今日のミッション一覧
- 達成 / 未達成 / 未回答の状態
- 今日の達成率
- カレンダー形式での日ごとの勝敗表示

###### 月次ミッションの可視化
- 今月のミッション一覧
- 目標値と現在値
- 進捗率
- 達成済みかどうか



# 非機能要件
1. オフライン動作（外部サーバーの使用なし）とする。通信環境がない状態でも、ミッションの確認・記録・分析ができるものとします。

2. SwiftDataを使用して、以下のデータを端末内に保存します。
- ミッション設定
- 日次ミッションの実行ログ
- 月次ミッションの進捗

3. App Storeへの公開は行わず、個人のiPhoneでの利用を前提としてます。

4. 初回起動時または通知設定時に、ユーザーへ通知許可を求めます(通知が許可されていない場合でも、アプリ内でミッション確認・記録は可能)


# DB設計

本アプリでは、SwiftDataを使用して以下の3つの主要モデルでデータを管理します。

- MissionConfig
- DailyMissionLog
- MonthlyMissionProgress

## 1. MissionConfig



### データベース設計
本アプリでは、SwiftDataを使用して以下のモデルでデータを管理します。

```swift
enum RepeatType: String, Codable {
    case once
    case daily
    case weekly
}
```
RepeatTypeは、繰り返し周期を定義するenumで、String/Codable対応によりJSONとの変換や保存を簡単に行えるようにしました。

```swift
enum MissionResult: Int, Codable {
    case unanswered = 0
    case success = 1
    case failure = 2
}
```
MissionResult は、ミッション結果を3択で管理するための型です。
各値の定義は以下とします。
unanswered = 0：未回答
success = 1：達成
failure = 2：失敗

```swift
@Model
final class MissionConfig {
    @Attribute(.unique) var id: UUID
    var title: String
    var category: String
    var repeatType: RepeatType
    var repeatWeekdays: [Int] // 1=月曜 ... 7=日曜、など明記
    var hour: Int
    var minute: Int
    var isMonthly: Bool
    var isEnabled: Bool
    var createdAt: Date
}
```
MissionConfig は、ミッションの設定内容を保存するSwiftDataモデルです。

```swift
@Model
final class MissionLog {
    @Attribute(.unique) var id: UUID
    var mission: MissionConfig?
    var configIDSnapshot: UUID
    var titleSnapshot: String
    var scheduledDate: Date
    var scheduledHour: Int
    var scheduledMinute: Int
    var result: MissionResult
    var respondedAt: Date?
}
```

#### 3. 統計・分析ロジック
* 達成率: (全成功ログ数 / 全回答ログ数) * 100で算出し表示する
* 継続日数: 今日のログから過去へ向かって、result == 1 が連続している個数をカウント
* 魔の時間帯（傾向）: 失敗ログ（result == 2）を targetTime の時間(Hour)ごとに集計し、件数が多い順に表示する
