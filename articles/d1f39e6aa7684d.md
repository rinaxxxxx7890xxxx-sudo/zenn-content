# はじめに
Zennで技術記事を書く流れを、GitHubとVS Codeを使ってローカル管理する手順として整理します。
「GitHubで管理 → VS Codeで執筆 → pushで公開」という流れです。
---

## 1. GitHubアカウント作成

まずGitHubのアカウントを作成します。
1:GitHub(https://github.com/)にアクセス
2:アカウント登録（メール・ユーザー名・パスワード設定）
3:メール認証を完了
ここまででリポジトリを作る準備が完了です。

---

## 2. Zenn用リポジトリを作成

Zennの記事はGitHubリポジトリと連携して管理します。

【リポジトリ作成手順】
1:GitHubで「New repository」をクリック
2:リポジトリ名を決める（例：zenn-content）
3:Publicで作成（Zenn連携のため）
※READMEは任意でOK

---

## 3. ローカルにクローンする

作成したリポジトリをローカルに持ってきます。

```bash
git clone https://github.com/ユーザー名/zenn-content.git
```

⚠️ よくあるエラー
```bash
Repository not found
```

この場合の原因はほぼ以下です：
URLが間違っている
privateリポジトリをcloneしている
ユーザー名・リポジトリ名の打ち間違い

→GitHubの「Code」ボタンからURLをコピーするのが確実です。
![リポジトリ作成画面](/images/evidence01.png)

---

## 4. Zenn CLIを使う（ローカル開発を快適にする）

Zennには公式CLI（zenn-cli）があり、ローカル環境で記事執筆を効率化できます。

### 主なメリット

- 記事の雛形を自動生成できる
- ローカルでプレビュー確認ができる
- GitHub連携と相性が良い
- 画像パスや構文ミスを事前にチェックできる

---

### インストール

```bash
npm init --yes
npm install zenn-cli
````

---

### 初期化

```bash
npx zenn init
```

以下の構成が作成されます：

```
articles/
books/
```

---

### 記事作成

```bash
npx zenn new:article
```

→ タイトルやslug付きのテンプレートが自動生成されます。

---

### プレビュー

```bash
npx zenn preview
```

→ ローカルで記事の表示確認ができ、公開前のミス防止になります。

---

## 5. ZennでのMarkdown記法のポイント

ZennはMarkdownベースですが、書き方を意識すると読みやすくなります。

---

### 見出しルール

* `#`：記事タイトル（1つだけ）
* `##`：章レベル
* `###`：補足・詳細

→ 構造を揃えることで読みやすくなります。

---

### コードブロックは必ず言語指定

```bash
```bash
git push origin main
```


```php
```php
echo "Hello Zenn";
```


```js
```js
console.log("Hello");
```


→ シンタックスハイライトが有効になり可読性が上がる

---

### 画像の書き方

Zennでは必ずルートパスを使う

```md
![説明](/images/sample.png)
```

---

### 改行のルール

* 空行 → 段落になる
* 改行だけ → 反映されない

---

## 6. VS Codeで開く

クローンしたディレクトリをVS Codeで開きます。
```bash
cd クローンしたフォルダ名
code .
```

---

## 7. Zenn記事フォルダ構成を作成

Zennでは基本的に以下構成で記事を書きます。
```bash
zenn-content/
 ├── articles/
 └── books/
```

記事作成例
```bash
articles/20260507_wordpress_trouble.md
```

---

## 8. 記事を書く（Markdown）

Zenn記事はMarkdownで書きます。

記述例：
```bash
---
title: "WordPress開発で詰まったポイントまとめ"
emoji: "📝"
type: "tech"
topics: ["WordPress", "PHP", "初心者"]
published: true
---

# はじめに

この記事では...
```

---

## 9. Gitでコミット & push

```bash
git add .
git commit -m "Zenn記事追加"
git push origin main
```

---

## 10. Zennに反映される

GitHubにpushすると、自動でZenn側に同期されます。
少し待つと記事一覧に表示されます。

---

## まとめ

Zennでの執筆フローは以下の通りです。

1. GitHubアカウント作成
2. リポジトリ作成（zenn-content）
3. ローカルにclone
4. VS Codeで記事作成
5. Markdownで執筆
6. git pushで公開

---

## 補足（今回のポイント）

今回のやり取りで重要だったのはここです：

- `Repository not found` → URLミスが原因になりやすい
- Zenn記事は「articlesフォルダ管理」
- VS Codeで編集 → Git pushが基本フロー
