# はじめに
Zennで技術記事を書く流れを、GitHubとVS Codeを使ってローカル管理する手順として整理します。
「GitHubで管理 → VS Codeで執筆 → pushで公開」という流れです。
---

## 1. GitHubアカウント作成

まずGitHubのアカウントを作成します。
1:GitHub(https://github.com/)にアクセス
2:アカウント登録（メール・ユーザー名・パスワード設定）
3:メール認証を完了
ここまででリポジトリを作る準備が完了です。

---

## 2. Zenn用リポジトリを作成

Zennの記事はGitHubリポジトリと連携して管理します。

【リポジトリ作成手順】
1:GitHubで「New repository」をクリック
2:リポジトリ名を決める（例：zenn-content）
3:Publicで作成（Zenn連携のため）
※READMEは任意でOK

---

## 3. ローカルにクローンする

作成したリポジトリをローカルに持ってきます。

```bash
git clone https://github.com/ユーザー名/zenn-content.git
```

⚠️ よくあるエラー
```bash
Repository not found
```

この場合の原因はほぼ以下です：
URLが間違っている
privateリポジトリをcloneしている
ユーザー名・リポジトリ名の打ち間違い

→GitHubの「Code」ボタンからURLをコピーするのが確実です。
![リポジトリ作成画面](/images/evidence01.png)

---

## 4. VS Codeで開く

クローンしたディレクトリをVS Codeで開きます。
```bash
cd クローンしたフォルダ名
code .
```

---

## 5. Zenn記事フォルダ構成を作成

Zennでは基本的に以下構成で記事を書きます。
```bash
zenn-content/
 ├── articles/
 └── books/
```

記事作成例
```bash
articles/20260507_wordpress_trouble.md
```

---

## 6. 記事を書く（Markdown）

Zenn記事はMarkdownで書きます。

記述例：
```bash
---
title: "WordPress開発で詰まったポイントまとめ"
emoji: "📝"
type: "tech"
topics: ["WordPress", "PHP", "初心者"]
published: true
---

# はじめに

この記事では...
```

---

## 7. Gitでコミット & push

```bash
git add .
git commit -m "Zenn記事追加"
git push origin main
```

---

## 8. Zennに反映される

GitHubにpushすると、自動でZenn側に同期されます。
少し待つと記事一覧に表示されます。

---

## まとめ

Zennでの執筆フローは以下の通りです。

1. GitHubアカウント作成
2. リポジトリ作成（zenn-content）
3. ローカルにclone
4. VS Codeで記事作成
5. Markdownで執筆
6. git pushで公開

---

## 補足（今回のポイント）

今回のやり取りで重要だったのはここです：

- `Repository not found` → URLミスが原因になりやすい
- Zenn記事は「articlesフォルダ管理」
- VS Codeで編集 → Git pushが基本フロー
